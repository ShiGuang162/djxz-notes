# 微信小程序个人博客项目计划

## 一、项目概述

### 1.1 项目背景
开发一个微信小程序个人博客，用于记录日常生活点滴，包含打卡功能。

### 1.2 核心需求

| 需求类别 | 具体功能 |
|---------|---------|
| 内容类型 | 图文（文字+图片） |
| 打卡功能 | 每日打卡、目标打卡、连续奖励 |
| 数据存储 | 微信云开发、用户登录 |
| 社交互动 | 点赞、评论 |

---

## 二、技术架构

### 2.1 技术栈
- **框架**: 微信小程序原生框架
- **后端**: 微信云开发（CloudBase）
- **数据库**: 云数据库（NoSQL）
- **存储**: 云存储

### 2.2 文件结构

```
├── app.js                 # 小程序入口
├── app.json               # 全局配置
├── app.wxss               # 全局样式
├── pages/                 # 页面目录
│   ├── index/             # 首页
│   ├── diary/             # 日记详情/编辑
│   ├── checkin/           # 打卡页面
│   ├── profile/           # 个人中心
│   └── comments/          # 评论页面
├── cloudfunctions/        # 云函数
│   ├── login/             # 用户登录
│   ├── diary/             # 日记相关
│   ├── checkin/           # 打卡相关
│   └── comment/           # 评论相关
├── utils/                 # 工具函数
└── components/            # 自定义组件
    ├── diary-card/        # 日记卡片
    ├── checkin-calendar/  # 打卡日历
    └── comment-list/      # 评论列表
```

---

## 三、页面规划

### 3.1 页面列表

| 页面路径 | 页面名称 | 功能描述 |
|---------|---------|---------|
| `/pages/index` | 首页 | 展示日记列表、打卡状态 |
| `/pages/diary/detail` | 日记详情 | 查看单篇日记、点赞评论 |
| `/pages/diary/edit` | 写日记 | 创建/编辑日记 |
| `/pages/checkin/index` | 打卡页面 | 打卡日历、连续天数、目标管理 |
| `/pages/profile/index` | 个人中心 | 用户信息、统计数据 |

### 3.2 页面流程图

```
首页 ──> 日记详情 ──> 点赞/评论
  │
  └──> 写日记
  │
  └──> 打卡页面
  │
  └──> 个人中心
```

## 四、数据库设计

### 4.1 集合结构

#### 1. users（用户表）
| 字段名 | 类型 | 说明 |
|-------|------|------|
| _openid | string | 用户唯一标识 |
| nickName | string | 用户昵称 |
| avatarUrl | string | 头像URL |
| createTime | timestamp | 创建时间 |

#### 2. diaries（日记表）
| 字段名 | 类型 | 说明 |
|-------|------|------|
| _id | string | 日记ID |
| userId | string | 用户ID |
| content | string | 日记内容 |
| images | array | 图片URL列表 |
| likes | number | 点赞数 |
| likedUsers | array | 点赞用户ID列表 |
| createTime | timestamp | 创建时间 |
| updateTime | timestamp | 更新时间 |

#### 3. checkins（打卡表）
| 字段名 | 类型 | 说明 |
|-------|------|------|
| _id | string | 记录ID |
| userId | string | 用户ID |
| checkDate | date | 打卡日期 |
| goalId | string | 目标ID（可选） |
| createTime | timestamp | 创建时间 |

#### 4. goals（目标表）
| 字段名 | 类型 | 说明 |
|-------|------|------|
| _id | string | 目标ID |
| userId | string | 用户ID |
| title | string | 目标名称 |
| description | string | 目标描述 |
| targetDays | number | 目标天数 |
| completedDays | number | 完成天数 |
| status | string | 状态（active/completed） |
| createTime | timestamp | 创建时间 |

#### 5. comments（评论表）
| 字段名 | 类型 | 说明 |
|-------|------|------|
| _id | string | 评论ID |
| diaryId | string | 日记ID |
| userId | string | 用户ID |
| content | string | 评论内容 |
| createTime | timestamp | 创建时间 |

## 五、功能模块设计

### 5.1 用户登录模块
- 微信授权登录
- 用户信息存储
- 自动登录验证

### 5.2 日记模块
- 创建日记（文字+多图）
- 编辑日记
- 删除日记
- 日记列表展示
- 日记详情查看

### 5.3 打卡模块
- 每日打卡
- 连续打卡统计
- 目标管理（创建/删除目标）
- 目标打卡关联

### 5.4 社交模块
- 点赞功能
- 评论功能

## 六、开发计划

### 6.1 阶段划分

| 阶段 | 时间 | 任务 |
|------|------|------|
| 第一阶段 | 1-2天 | 项目初始化、云开发配置、登录模块 |
| 第二阶段 | 2-3天 | 日记模块（CRUD） |
| 第三阶段 | 2-3天 | 打卡模块（日历、目标） |
| 第四阶段 | 1-2天 | 社交模块（点赞、评论） |
| 第五阶段 | 1天 | 测试、优化、发布准备 |

### 6.2 预期交付

- 完整的微信小程序代码
- 云开发配置完成
- 数据库集合创建完毕
- 可正常运行的功能

## 七、风险评估

| 风险点 | 影响 | 应对措施 |
|--------|------|---------|
| 云开发配置复杂 | 开发进度延迟 | 提前学习云开发文档 |
| 图片上传处理 | 性能问题 | 限制图片数量和大小 |
| 打卡日期计算 | 逻辑错误 | 编写单元测试 |

1. 遵循微信小程序开发规范
2. 注意用户隐私数据保护
3. 优化小程序加载性能
4. 确保良好的用户体验
